Section Seven: Dictionaries

Lesson Seventeen: The Dictionary Data Type

  • A dictionary is like a list, in the fact that it stores many different types of data, and it can hold multiple data types at once as well.
  • A dictionary uses a key-value pair, to hold this information. A key is much like an index, where when you refer to the key, it acts as the information that is stored within the key.
  • In Python, dictionaries are stored inside curly braces. Dictionaries, like all data types, can be stored inside variables.
  • Here's an example dictionary that you can write in the IDLE window: "myCat = {'size': 'fat', 'color': 'gray', 'disposition': 'loud'}"
  • You may have noticed, but all key-value pairs are separated by commas. For this particular dictionary, the keys are 'size', 'color', and 'disposition', while the values associated with the keys are 'fat', 'gray', and 'loud'.
  • You can access a keys value just like you would with a list's index. For this particular variable, if you wanted the value of the 'size' key, you would type: "myCat['size']". The value 'fat' would then be displayed onto the IDLE window.
  • Lists can also have integers as keys, and they can be whatever you want them to be, they don't have to start at zero.
  • Unlike lists, order of key-value pairs doesn't matter. If you wrote a list, and it's values were mixed up, but still the same, using the equal to operator would evaluate to False because for lists, order matters. If you had a dictionary with the same key-value pairs, but mixed up, using the equal to operator would evaluate to True.
  • Dictionary might be a bad name on account of the fact that it doesn't matter the order of the keys in it, but you can relate the Python dictionary to a real one by thinking of the key-value pairs as the word and it's definition. If you know the word, you can access the defintion, if you know the key, you can access the value.
  • You can use the in and not in operators to determine if a key is in a dictionary or not. This can be useful in attempting to prevent a KeyError which is what you get when you refer to a key in a dictionary that doesn't exist.
  • Like lists, dictionaries are immutable, meaning that if you store them in a variable, the variable keeps track of a number associated with the dictionary, not the information inside the dictionary itself.
  • There are 3 methods that you can use to access various information about a dictionary.
    1. The first is the keys method which looks like this: "list(variablename.keys())". This method will return the keys held inside the dictionary.
    2. The next method is the values method, which is the same as the last example, except you swap the word "keys" for "values". This method will return all the values inside of a dictionary.
    3. The final method is the items method, and it is also the same as thelast 2, except the word before the last parenthesis is "items". This method will spit out a list with 2-item tuples(which is basically the same thing as a list) as the items. Inside the tuples are the key and value pairs, where the key is always listed first, and the value always listed second.
  • You can use for loops to iterate through the output of each method. Just simply write "for k/v in variablename.keys():

    print(k/v)

    "
  • If you want to use a for loop on the items method, simply use 2 variable separated by a comma for both the declaration of the for loop, and when printing the for loop.
  • Python offers a get() method, to help you avoid KeyErrors and the need to write tedious if statements. The get method looks like this: "variablename.get('keyname, defaultvalue')"
  • There is also a method called setdefault(), which is essentially the opposite of the get method. The setdefault() method is used to set a value for a key, if it doesn't already exist, whereas the get() method uses the value associated with a key, and if it doesnt exist, it uses the default value.
  • You can use the setdefault() method to create a program that counts the occurences of a letter in a string. The program looks like this: "message = 'It was a cold bright day in April, and the clocks were striking thirteen.'
    count = {}

    for character in message.upper():

    count.setdefault(character, 0)

    count[character] = count[character] + 1

    print(count)". You can save this program as "characterCount.py".
  • This program starts with a string variable, which could actually be set to be an input function to make it a little more interactive. Then we initialize the dictionary and name it count. Next we create a for loop, and for each iteration, if there's a character that hasn't appeared yet, it will set the default to 0, and then add one upon each appearance in the string.
  • You can use the pprint module to make the output of this program much more organized. Simply write import pprint at the top of the file, and then where it says "print(count)", replace it with "pprint. pprint(count)", and the output will be organized in a column and ordered alphabetically.

Lesson Eighteen: Data Structures

  • You can use data structures like dictionaries and lists to make complex things. This lesson mostly covered some parts of a program using dictionaries to represent a game of tic-tac-toe.
  • The type function can identify the type of data you are working with. Just type "type()" and whatever data you put inside, it will output the type of data you inputted. You can pass in strings, integers, floats, and even variables, indexes and keys.
