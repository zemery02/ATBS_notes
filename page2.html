Section Two: Flow Control

Lesson Four: Flow Charts and Basic Flow Control Concepts

  • In a flow chart, you start a designated point, and follow the arrows until you get to the desired finishing point. There will be a link to a website that explains this concept in the section resources part below.
  • Instructions called "flow control statements" can decide which Python functions to execute under which conditions.
  • To understand how to make a flow chart-like program in Python, we must first learn the concept of Boolean Values, Comparison Operators and Boolean Operators.
  • The Boolean data type has only two values, True, and False. You can store boolean values in variable just like integers, strings, and floats. Just make sure to capitalize the first letter of True or False, and only the first letter.
  • Next we have comparison operators. There are 6 total comparison operators, and they can all be used to in the place of a arithmetic function. The six operators are:
    1. "==" Equal to
    2. "!=" Not equal to
    3. "<" Less than
    4. ">" Greater than
    5. "<=" Less than or equal to
    6. ">=" Greater than or equal to

  • So, with all that in mind, if you were to put 42 == 42 into Python, what do you think would be returned? If you said "the boolean value of True", you'd be correct, and you can give yourself a pat on the back. Same logic applies to 42 == 'Hello' evaluating to False.
  • Theres also "!=", not equal to, that evalutates to True when a statement like, "2 != 3" is made. All of the less than, and less than or equal to statements work just as you'd expect them to as well.
  • You can also make these as complicated as you would with arithmetic, you could assign a value, like 17 to a variable called "myAge" and then say "myAge < 21", which would evaluate to True.
  • In Python, integers and strings will never be equal to each other. For instance, the statement 42 == '42', would evaluate to False, because the first 42 is an integer, and the next one is a string.
  • Float values and integers however, can evaluate to True in one of these comparison statements.
  • It's important to note that "==" and "=" are not the same thing! "=" is the assignment operator, and it is used for assigning values to variables, while "==" is strictly a comparison operator that returns boolean values if formatted properly.
  • There are also 3 boolean operators, "and", "or", and "not"
  • The and operator evaluates to True only if the 2 conditions given are both True. The and operator is just the word "and" all lower-case.
  • The or operator evaluates to True if one or both of the conditons given are True. The or operator is just the word "or" all lower-case.
  • The not operator evaluates to the opposite boolean value given. The not operator can only be given one statement, and it is also just the word "not" all lower-case.
  • You'll often mix boolean and comparison operators together in the same statement. For instance you could write:

    myAge = 17
    myPet = 'dog'
    myAge < 89 and myPet = 'dog'
    This statement evaluates to True.

Lesson Five: If, Else, and Elif Statements

  • Now that we know about boolean values as well as comparison and boolean operators, we can now cover some flow control statements. The most common of these are "if", and "else".
  • To start this lesson go ahead and type the following code into a new text editor window:
    "name = 'Alice'
    if name == 'Alice':

    print('Hello Alice')

    print('Done')"
    You can save this as "if_example.py".
  • The expression in the if statement, "if name == 'Alice'", is known as a condition.
  • If the condition evaluates to True, the indented code will will run. Since, in this example, name does equal 'Alice' it prints out the greeting.
  • If the conditon is False, it just skips the indented code, and in this example, just prints "Done".
  • The indented part of the code is known as a block, the indentation is the way to let Python know what is part of the if statement, and what is not.
  • If needed, you can have a block inside of a block. Blocks are also sometimes known as "clauses", just in the same way conditions are sometimes known as expressions.
  • As well as indentation, new blocks also require a colon at the end of the previous line.
  • We can use an else statement if we want something to happen if a condition is evaluated to False.
  • Open a new text file and write the following code:
    password = 'swordfish'
    if password == 'swordfish':

    print('Access Granted.')

    else:

    print('Wrong Password.')

    You can save this file as "if_else_example.py".
  • In this example, one and only one block will be executed by Python. For now it will print "Access Granted.", but if we changed the password variable to be "rosebud", it would print "Wrong Password."
  • If you need to write a statement with more than one condition, you can use the elif statement, which is just an abbreviation of "else if".
  • To get a good idea of the elif statement, input the following code into a new text editor window:
    "name = 'Bob'
    age = 3000
    if name == 'Alice':

    print('Hello, Alice')

    elif age < 12:

    print('You are not Alice, kiddo.')

    elif age > 2000:

    print('Unlike you, Alice is not an undead, immortal vampire.')

    elif age > 100:

    print('You are not Alice, grannie.')"

    You can name this file "if_elif_example.py".
  • You can have as many elif statements follow an if statement as you need, but order does matter, because your program will ignore anything after a condition that is evaluated as True.
  • So in this example, it stops running after we get to the condition "elif age > 2000", and executes whatever is written for that condition.
  • It is also common to add an else statement to the end of an elif statement, so that something happens even if everything is evaluated to False.
  • You could ask for an input, and store into a variable, then you could write and if else statement that just says "if variablename:" this only evalutes to True if the user stores a value inside of variable by inserting something into the input function. These statements are known as "Truthy" and "Falsey" values.
  • It is more practical to have a condition like "if variablename != ''" because it makes a condition for Python to check, in this case, it's checking to see if variablename is an empty string or not.
  • For integers, 0 is Falsey, and all other values are Truthy. Likewise, 0.0 is the only Falsey float value.
  • Like the str and int function, you can use the bool function to turn a string, integer, or float value into a boolean value. For example you could write into the IDLE window, "bool(0)", and it would e valuate to false.

Lesson Six: While Loops

  • The next flow control statement we'll cover is the while loop, which executes a block of code repeatedly so long as the given conditions are True.
  • You can open a new text editor window and enter the following code to follow along:
    "spam = 0
    while spam < 5:

    print('Hello World!')

    spam = spam + 1"

    You can save this file as "while_example.py"
  • When run, this code prints out the string "Hello World" five times. This is because the spam variable starts at 0, and adds one everytime the block of code is executed. So the spam variable starts at zero and runs until it gets to five, so at the end, spam = 4.This process of adding one to every loop is called incrementing.
  • When the loop executes the block of code, it is called an iteration.
  • If you were to enter the same block of code into a if statement, it would be much harder to come to the end of the statement because you'd have to call the statement several times, because the if statement does not loop back to the start of the statement and check if the conditions are True before running again.
  • Open up a new text editor file and write the following code:
    "name = ''
    while name != 'your name':

    print('Please type your name.')

    name = input()

    print('Thank you!')"
    You can save this file as "yourName.py"
  • This condition starts out by setting the name variable to a blank string so that the condition "name != 'your name'" is True, and the code inside the loop is executed.
  • This loop will run forever unless you were to literally type out "your name", when prompted to provide input.
  • A loop is good way to continually ask the user for input, until the information they give is valid. So if you were to ask for a user's age, you could use a loop to make sure that the user does not provide an answer with a negative number, or a string like 'Abraham Lincoln'.
  • If you were to ever write an infinite loop by setting your condition to "while True", and need to back out of the code, simply press CTRL + C, and a keyboard interupt error will stop the infinite loop that will eventually crash your computer.
  • One statement you can use inside of a loop is the "break" statement, which instantly stops your code from running.
  • You could change the yourName.py example to demonstrate this statement. First delete the conditions and insert "while True:", next, while still inside your block of code, write an if statement that has these conditions: "if name == 'your name':", and inside of the if statement, write "break".
  • This does not change anything about the code, it is just an example of how you could use the break statement.
  • Like break statements, "continue" statements are also used inside of loops.
  • Heres a simple program that you can write out to have an example of the continue statement:
    "spam = 0
    while spam < 5:

    spam = spam + 1
    if spam == 3:

    continue

    print('spam is ' + str(spam))"

    You can name this file "continue_example.py".
  • The output of this code is "spam is 1, spam is 2, spam is 4, spam is 5". It skips 3 because on the iteration where spam = 3, the if statement is True, and the contiue statement is executed. The continue statement stops the execution and goes back to the start of the loop, so in this case, the print function in the loop is skipped.
