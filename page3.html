Section Three: Functions

Lesson Eight: Python's Built-in Functions

  • All Python programs have a basic set of functions known as "built-in functions", and they include but are not limited to, the print function, the input function, and the len function.
  • Python also comes with a set of modules known as "the standard library", and it contains modules that contain specific functions. For instance the Math module has mathematic related functions, the random module has functions related to generating and dealing with random numbers, and there are many more not mentioned.
  • When you want to use one of these modules, you must use an import statement. You can do this in the IDLE window by typing "import random", and on the next line, you can now use a function from this random module. In the lesson, it's instructed to type "random.randint(1, 10)", which gives us a random number between 1 and 10 (including both 1 and 10).
  • Since randint isn't a built-in function, you must type "random." before you use any functions from the random module.
  • Another way you can import a module is by typing from modulename import *, where the "*" is equal to everything in the module. This allows you to omit the "random." before every use of a function from the random module, and the many other modules included in the Python standard library.
  • While using the second import method is easier, using the first one makes for more readable code, since you know where a function like randint is coming from.
  • You can use a function from the "sys" module called "exit", to terminate your program early if you need, it looks a little something like this, feel free to type this into a new text editor window if you'd like to follow along:
    "import sys
    print('Hello!')
    sys.exit()
    print('Goodbye!')"
    You can name this file "sysexit.py".
  • You can also use third-party modules, but before you import, you must use the Pip program, which is also included in Python. You have to use the Pip program from the command line, also known as the terminal.
  • Consult the section resources part of this page to check out how to install third party modules. Now, install the pyperclip module using the instructions.
  • The pyperclip module allows you to copy and paste text to and from your clipboard, using the pyperclip.copy(), and pyperclip.paste() functions.
  • You can go ahead and import the pyperclip module in the IDLE window and type "pyperclip.copy('Hello world!')". "Hello World" is now copied to your clipboard, and you can use it whereever you'd like.
  • In the next line, you can type out "pyperclip.paste()" and it will copy the last thing on your clipboard, which should be "Hello World", assuming you didn't get distracted in the middle of this lesson.

Lesson Nine: Writing your own Functions

  • Since functions are just mini-programs inside of other programs, you can build your own using everything that you've learnt so far.
  • You can write the following code to create your first function:
    "def hello():

    print('Howdy!')

    print('Howdy!!!')

    print('Hello there.')

    hello()
    hello()
    hello()""
    You can name this file "example.py".
  • So the def statement only defines a function, and does not execute the code inside. This is why we typed "hello()" three times at the bottom of the document. This process is known as "calling" your function. Although you don't have to write it, because it's part of Python's standard library, everytime you use the print function, you are calling it.
  • The main purpose of functions is to group together code that is run together multiple times. If you don't do this, you'd have to copy and paste the code block everytime you need to use it. Copy and pasting is generally discouraged because if you want to update the code, or if you need to squash a bug, you might have to look through a lot more code than you would have needed to if you just used a function.
  • You use arguements everytime you call the print or len functions. These arguements are always pleaced in between parenthesis. You might have noticed the parenthesis when we defined and called our hello function. You can update your code to use an arguement by placing a variable name in the line inside the parenthesis where you defined the statement, for now, make the variable name, "name". Then, replace everything in the block of code with one print function that reads "print('Hello ' + name)". Now you can call your function and insert a name, which might look like "hello('Zac')". There you have it, your first written function can now take an arguement.
  • It's important to note that in this context, your name arguement is known as a "parameter". To clear up any confusion, the string "Zac", that is passed in is known as an argument, while the placeholder variable "name" is known as a parameter.
  • You can use functions in between your code, just as you would a variable or any other data type. For instance, you can use the len function and create the string "hello has " + str(len('hello')) + ' letters'. This function evaluates to the string "Hello has 5 letters".
  • You can use a return statement to specify what the function should evaluate to.
  • To demonstrate, go ahead and open up a new text editor file and type the following code:
    "def plusOne(number):

    return number + 1

    newNumber = plusOne(68)
    print(newNumber)"
    You can name this file "plusOneExample.py".
  • The None value is the only value in the None data type. If you were to type the statement "spam = print()", the IDLE window would return nothing, because print without an arguement evaluates to the None data type. This means that every function call has a return value.
  • If you don't use a return statement in your functions, it always returns the none value.
  • There are arguements known as keyword arguements that you can use in some of the functions that are in Python's standard library. There is the end and sep keywords for print. The end keyword is useful for times where you have multiple print statements back to back. The print function automatically assigns a newline character after it has been executed, but you can add a second arguement at the end of your string in the parenthesis that looks like: "end=''". Now the two print statements will be right next to each other with no spaces.
  • You can also use the sep keyword to separate words or phrases that are passed into a single print statement. Regularly, the statement "print('cat', 'dog', 'mouse')", would evaluate to "cat dog mouse", but if you wrote "sep=ABC" inside the parenthesis, next to 'mouse', the print statement would evaluate to "catABCdogABCmouse".
  • Most of the time you won't need to use these keyword arguements, but they can help you format the print function exactly how you want it to.
