Section Four: Handling Errors with try/except

Lesson Eleven: Try and Except statements

  • With the knowledge you have right now, if you were to come across an error, your entire program would crash.
  • This lesson will show you how to have your program detect errors, hendle them, and then continue to run.
  • You can input the following code to follow along with the lesson:
    "def div42by(divideBy):

    return 42 / divideBy

    print(div42by(2))
    print(div42by(12))
    print(div42by(0))
    print(div42by(1))"
    You can name this program "try-except_example.py"
  • If you run this code, you will get the first 2 answers, 21.0, and 3.5, but then you will get an error called "ZeroDivisionError". This is because it is impossible to divide by zero, and so the program spits out an error instead of spazzing out.
  • Now you can modify your code to look like this:
    "def div42by(divideBy):

    try:

    return 42 / divideBy

    except ZeroDivisionError:

    print('Error: You tried to divide by zero.')

    print(div42by(2))
    print(div42by(12))
    print(div42by(0))
    print(div42by(1))"
    Now you can resave the program under the same name.
  • Now the output prints the error message, and since there is nothing to return, it still prints "None", and then it continues on and prints 42.0, which is the answer to 42 / 1.
  • You can also omit the type of error in the except statement, and it will run exactly as it did before.
  • Here's another example of a try/except statement, feel free to follow along:
